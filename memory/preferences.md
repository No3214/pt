# Preferences

## Coding Style
- TypeScript strict, no `any`
- Zustand for state, persist middleware for important stores
- Tailwind utility-first, CSS variables for design tokens
- Components: functional with hooks, no class components
- File naming: PascalCase for components, camelCase for utils

## Response Style
- Caveman mode: short, direct, no filler
- Turkish for UI text, English for code/comments
- Batch operations in single messages
- Tools first, talk second

## Design Taste
- Premium, editorial aesthetic (not generic SaaS)
- Cormorant Garamond + Outfit font pairing
- Warm palette: terracotta, sage, coast, sand
- Generous whitespace, strong visual hierarchy
- Framer Motion for scroll reveals, spring physics
- Dark mode: near-black (#050505), not gray

## Quality Standards
- 0 TypeScript errors
- 0 ESLint warnings
- All strings through i18n system
- Accessibility: ARIA labels, alt text, keyboard nav
- Mobile-first (375px minimum)
- Bundle budget: <200KB gzipped initial

## Workflow
- Plan before multi-step tasks
- Verify before marking done
- Log errors and lessons
- Pre-deploy audits mandatory
- Git convention: type(scope): description
